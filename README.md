# 小狗一起拉屎好吗？

## 无代码协作更新（Issue 投稿）

现在可以通过 GitHub Issue 表单来新增编年史事件，不需要直接修改代码：

1. 打开仓库的 `Issues` 页面，点击 `New issue`。
2. 选择 **新增编年史事件** 模板。
3. 按表单填写事件日期、标题、描述、相关小狗（支持逗号/换行分隔）。
4. 提交后会自动触发 GitHub Action：
   - 把内容写入 `src/data/chronicle.json`
   - 自动创建或更新一个 PR
   - 在原 Issue 下留言 PR 链接
5. 管理员审核并合并 PR 后，GitHub Pages 会自动更新站点。

### 可选图片

如果事件有图片，请先把图片放到仓库 `public` 目录下，再在表单中填写图片路径（例如 `/logs/20260330/new.jpg`）。
